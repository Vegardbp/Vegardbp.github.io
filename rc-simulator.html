RC Car Simulator

C++ · threepp · Bullet Physics · Git · Python

A hardware-accurate simulation of a Sphero RVR RC car. Indistinguishable from the real system to external control systems, other than IP-address.

Developed as part of a group project focused on cyber-physical systems.

I was responsible for the entire simulator (Including Models and Textures) and control system, while the others were responsible for cross-program communication systems, physical hardware, and GUI.

Graded B, with the reason that the report was on C level, but good simulation, and monocular slam attempt, made the project overall a B.

Source code on GitHub

Details

Hardware-accurate Simulation is indistinguishable from the real Sphero RVR to any external controller or GUI
Bullet Physics Full collision detection and vehicle dynamics via Bullet physics engine
Live video stream 3D first-person view streamed over UDP-IP to controller and GUI in real time
C++ / threepp 3D rendering via threepp (C++ port of three.js), no game engine
Monocular Slam Custom monocular SLAM implementation in python without IMU data
Object Recognition Object recognition using OpenCV in python, able to recognise a dozen objects in the simulator, and more than 60 objects in the real world
Baked Lighting Custom baked ray-traced lighting for static scene despite weak lighting support in threepp
